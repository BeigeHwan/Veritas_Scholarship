1. Parameter-Efficient Fine-Tuning with Layer Pruning on Free-Text Sequence-to-Sequence Modeling

Yunqi Zhu, Xuebing Yang, Yuanyuan Wu, Wensheng Zhang

1. Abstract

Language Model의 발전은 더 효율적인 parameter tuning 방법에 대한 interest의 증가로 이어졌다. LoRA는 pre-trained model을 freeze 시키고 small-scale trainable parameter를 삽입함으로써 tuning을 이루어낸다. 이 방법론을 기반으로 본 논문에서는 LoRA와 structured layer pruning을 통합하는 framework를 제안한다. 이 framework는 두가지의 deidentified 의료 리포트 요약 관련 데이터와 두가지 public 의학 대화 데이터에서 테스트한다. 기존 모델의 parameter 중 단 0.6%만을 바꾸고, 그리고 30% 이상의 transformer-layer를 pruning 함으로써 본 framework는 50% 이상의 GPU memory 사용을 줄이고 training phase에서 100%의 성능 향상을 이루어낼 수 있다. 이러한 성능 향상은 92%의 seq2seq task의 generation quality를 보존하면서 이루어진다.

2. Introduction

Pre-trained Language Model(PLM)은 우월한 성능으로 인해 다양한 NLP Task들에 사용되고 있다. 그러나 모델의 크기가 커짐에 따라 full fine-tuning은 computationally expensive하고 time-consuming하기 떄문에 제한된다. 이러한 문제를 해결하기 위해 parameter-efficient fine-tuning 방법들이 각광받고 있다. 예를 들어 Adapter-based tuning은 작은 scale trainable neural network를 추가하고 나머지 변수들을 freeze 시킨다. 최근에는 adapter 기반의 variant인 low-rank adaptation(LoRA)가 small-scale parameter tuning에서 자주 사용되고 있다. LoRA는 nonlinear activation을 제거하여 학습시키는 기법이다. 또한 PLM의 pruning은 모델에 경량화에 도움이 된다. LoRA와 structured layer pruning(L-prun)을 결합하여 PLM에 적용시킨다. 이는 아래 도표에서 확인할 수 있다.

위 도표에서 보이듯이 pre-trained language model을 freeze시키고 LoRA를 feed-forward network에 inject한다. 이후 transformer layer는 번갈아가면서 제거시킨다. 따라서 본 논문의 목적은 유연하고 가벼운 small scale plug-in parameter들을 LoRA에 적용시키고 L-Prun을 통해 PLM을 경량화시키는 preliminary framework를 제안하는 것이다. 쉽게 말해 baseline을 제시하는 것.

Experiment의 순서를 최종적으로 정리하며 다음과 같다 :

1. Small-Scale parameter-tuning module LoRA와 L-Prun을 통합하는 프레임워크를 제시한다.

2. MIMIC-IV-Note에 기반해서 MIMIC-IV discharge 그리고 MIMIC-IV-radiology라는 deidentified된 report summarization 데이터 set을 만든다. 위 두가지 데이터와 두가지 다른 medical dialogue generation dataset에 실험을 진행한다.

3. 기존 PLM 모델의 parameter 중 0.6%를 업데이트하고 30% 이상의 transformer layer를 pruning 함으로써 학습 속도를 두배 빠르게 만들어주고50%의 gpu 사용량을 줄일 수 있다. Generation qualities on medical Seq2Seq tasks는 92%가 유지된다.

  1. Parameter-Efficient Fine-Tuning : Fine-tuning의 대표적 방법론은 크게 세가지로 나뉘어질 수 있다. Addition, Specification, and Reparameterization. Addition은 Adapter-based tuning기법으로써 lightweight down and up projection MLP들, nonlinear activation function을 삽입한다. 또한 prompt-based tuning으로 input을 freeze시키고 NLP task의 사전 정보를 활용하여 additional traiable conetext를 기존 input에 추가한다. Specification은 제한된 숫자의 layer, module, weights, and biases만 학습하는 방식이다. 추가적인 parameter가 사용되지 않기 때문에 가장 memory-efficient한 모델이다. Reparameterization : NLP 과제에 사용되는 PLM이 low intrinsic rank optimization(LoRA)을 통해 압축된 parameter들로 표현될 수 있다는 가정하에 진행되는 방법이다. LoRA는 adapter의 구조를 따르지만 non-linear activation부분을 제거한다.
  1. Parameter Pruning : Structured Pruning은 empirical하거나 heuristic한 방법을 따르며, unstructured pruning은 몇가지 사전에 정의된 threshold 혹은 몇가지 학습가능한 parameter들이 adaptively decide which MLP, attention block or the entire layer to be dropped.

4. Methodology

본 논문에서는 LoRA와 structured layer pruning을 합치는 hybrid framework를 제안한다. 이는 메모리 사용, 학습 속도, 그리고 모델 사이즈를 축소 시킬 수 있을 것으로 예상된다. 따라서 본 논문의 목적은 NLP 작업에 쓰이는 PLM의 크기를 L-Prun을 통해 줄이고, 유연하고 가벼운 small-scale plug-in trained parameter들을 LoRA를 통해 도입하여 preliminary framework를 제공하는 것이다.

LoRA의 parameter 개수는 기존의 PLM parameter의 개수보다 훨씬 작으며(<<), 이 parameter들은 모델에 inject된다. 또한 L-Prun 기법은 PLM의 parameter들을 더 작게(<) 줄여준다. 총 N개의 하위 task들이 있다고 했을 때, full fine-tuning은 N*Pm의 용량을 차지하나, LoRA가 사용되게 되면 N*Pm:LoRA + Pm 개의 parameter들이 사용되게 된다. 여기에 Pruning까지 적용되게 되면 N*Pprun:LoRA + Pprun개의 parameter가 사용된다. 이 때 Pprun:LoRA < Pm:LoRA이다. 즉, LoRA는 Pprun개의 parameter들은 상수로 고정해놓고 (결정해야되기 때문에 Pprun개가 더해짐), 나머지 N개의 task별로 Pprun:LoRA개의 변수들을 사용한다.

LoRA는 PLM의 weight들을 다 freeze 시키고 두 개의 학습 가능한 Wdown과 Wup의 layer들을 inject한다. 따라서 x hat의 값은 위와 같이 도출되게 되고 alpha 값은 scaling 상수이다.

LoRA는 adapter의 variant로 생각할 수 있다. 그러나 layer들 사이에 nonlinear activation이 추가되는 adapter 방법과는 다르게 LoRA는 freeze해놓은 weight와 LoRA를 통해 삽입된 변수들을 합함으로써 low inference latency를 유지시킬 수 있다. 이러한 up, down projection으로 LoRA는 parameter의 개수를 훨씬 줄일 수 있다. 실험 결과에 따르면 transformer layer들을 alternating fashion으로 지우는 것(layer pruning)이 모델 사이즈를 줄이고, 학습이나 추론 단계의 속도를 크게 향상 시킬 수 있다고 한다. 따라서 encoder와 decoder 부분의 transformer layer들을 대칭적으로 drop 시켰다

5. Experiments

실험은 다음의 4개의 data set에서 이루어졌다. MIMIC-IV-discharge, MIMIC-IV-radiology, HealthCareMagic, 그리고 iCliniq이다. MIMIC 데이터 두개는 MIMIC-IV-Note로부터 생성되었다. 그리고 뒤에 두개 데이터(Dialogue)를 위해서는 HealthCareMagic과 iCliniq 을 사용해서 의학 대화 생성을 하였다. 두 데이터 모두 medical consultation platform에서 크롤링한 환자와 의사의 대화를 담고 있다.

Dataset의 statistic은 위와 같다. train/eval/test는 80:10:10의 비율로 나뉘어졌고 sents와 words는 각각 input과 ouput에 포함되어 있는 평균 문장과 단어 개수들을 뜻한다. novel은 source와 ouptut을 비교했을 때 겹치지 않는 n-gram의 비율이며 n-gram은 n개의 인접한 단어들이다.

Implementation Details

실험은 pre-trained된 BART-large와 T5 large로 진행되었다. Hyperparameter rank r은 16, 그리고 alpha는 32가 사용되었다. BART-large와 T5-large모델의 각각 33%, 그리고 41%의 layer가 drop되었다. Optimizer는 AdamW optimizer를 사용하였고 learning rate는 original과 L-Prun의 경우 5*10^-5으로 설정하였고, LoRA는 10^-4로 설정했다. 또한 최대 source length를 1024, 그리고 512로 각각 summarization과 dialogue set에 적용했다. 0.3 Epoch마다 모델 평가를 진행했고 ROGUE-1 점수가 제일 높은 checkpoint를 test set에 사용했다.

Evaluation Metrics

Summarization의 성능은 ROUGE, BERTScore and SummaC으로 측정되었고 dialogue의 성능은 ROUGE, BERTScore and BLEU로 평가되었다.

위에는 모델들의 performance를 보여준다. LoRAff+ Lprun이 가장 합리적인 방법론인 것으로 보여진다. 빠른 fine-tuning, 낮은 GPU 사용량, 그리고 좋은 summrization quality를 보장한다.

이어서 experimental result를 살펴보자. L-Prun은 BART-large나 T5-large의 성능을 크게 감소시키지 않았다. 또한 LoRA와 L-Prun의 도입은 각각 68%, 75%의 gpu 메모리밖에 사용하지 않는다. Integration of LoRA와 L-Prun은 기존 대비 모델의 0.6%만을 학습시키고 33% 혹은 41%의 BART-large 혹은 T5-large의 transformer layer를 drop하면서 오직 8%의 ROUGE score와 BERTScore의 감소를 보였다.

dialogue generation이 훨씬 높은 %novel N-gram 값을 가진다. 따라서 dialogue generation의 난이도가 summarization보다 더 어렵다는 것을 추측할 수 있따. 그러나 이 방법론은 5% 이내의 소폭 성능 감소를 가져오고 GPU의 가속화와 메모리 절약은 summarization task를 할 때만큼 잘 이루어진다.

2. BrainGB: A Benchmark for Brain Network Analysis with Graph Neural Networks

Hejie Cui, Wei Dai, Yanqiao Zhu, Xuan Kan, Antonio Aodong Chen Gu, Joshua Lukemire, Liang Zhan, Lifang He, Ying Guo, Carl Yang

1. Abstract

뇌의 구조적 그리고 기능적 연결성에 기반한 신경 촬영법은 최근 가장 만연한 방법론으로 자리 잡았다. 이를 분석하기 위해 복잡한 네트워크 구조를 기하학적으로 모델링하고 분석하는 GNN 기법이 활용될 수 있다. Graph의 중요성과 필요성에도 불구하고 네트워크 구조에 적합한 그래프 구조 모델의 체계적인 연구가 이루어지지 않았다. 이러한 간의 간극을 채우기 위해 BrainGB 모델이 고안되었다. BrainGB는 뇌 네트워크 분석의 standard 모형이고, 기능적 그리고 구조적 neuro-imaging을 실행할 때 다양한 modal에서도 적용 가능하게 설계되었다.

2. Introduction

뇌의 여러부분들 간의 기능적 연결성은 시계열 데이터로 표현되며, 구조적 연결성은 gray matter의 물리적 연결로 표현된다.

  • gray matter이란?!

    gray matter(회백질)은 주로 뇌의 신경세포들이 모인 조직이다. 뇌의 주요 활동들은 대부분 gray matter(회백질)에서 발생한다. white matter(백질)은 gray matter(회백질) 사이를 연결하는 조직이다. 정보를 전달하는 통로 역할을 한다.

이 두가지 정보 모두 뇌 분석을 할 때 유용한 정보로 활용된다. Regular한 데이터들은 고전적 RNN이나 CNN을 사용할 수 있다. 그러나 Irregular한 데이터들은 non-euclidean 성격을 띄기에 고전적 방법을 사용하기가 쉽지 않기에 새로운 방법론이 필요했다.

현재 뇌 네트워크 분석은 일반적으로 두가지 단계로 이루어진다. 첫번째 단계에서는 뇌 네트워크 구조를 neuroimaging data를 통해 모델링한다. 그리고 두번째 단계에서는 모델의 연결성을 활용하여 환자의 clinical outcome을 분류하는데 사용한다.

데이터셋 D는 N개의 subject들로 이루어져 있다.

Gn은 subject n의 뇌구조를 뜻하고 yn은 subject의 예측치(disease 같은 clinical condition)를 의미한다. 데이터셋 D는 모든 subject n의 뇌 네트워크 Gn을 포함하고 있다. 모든 이들의 뇌구조는 ROI(Region of Interest)에 의해 이루어진 brain parcellation을 따른다. 즉, 사용되는 M개의 노드는 동일하며 차이는 M개의 brain region사이의 edge connection에서 이루어진다.

Edge weight W는 ROI들 간의 연결강도를 뜻한다. 모델의 ouput은 예측치 yhat_n이다.

해당 논문에서는 MRI로부터 추출된 데이터를 사용했다. 다양한 modality의 MRI 데이터를 사용해서 여러 다른 뇌 네트워크를 모델링할 수 있다. fMRI(functional MRI)는 기능적 연결성을 확인할 수 있게 해준다. 신경 활동에 따른 Blood Oxygenation Level을 측정함으로써 이를 이룰 수 있다. 반면, dMRI(diffusion-weighted MRI)의 경우 white matter를 측정하여 뇌의 구조를 탐색할 수 있게 돕는다.

3. Functional Brain Network Construction

ABCD(Adolescent Brain Cognitive Development) 데이터는 기능적 분석에 사용하고 PPMI는 구조적 분석에 사용하고자 하였다. 그러나 raw MRI 데이터는 불필요한 artifact들을 제거해야 하고, 데이터를 standard format으로 변환시키는 과정이 필요하다. 따라서 본 논문에서는 여러 modality들의 뇌 네트워크를 구성할 수 있는 general&standard preprocessing pipeline을 제공하고 있다.

위 테이블에서 왼쪽이 일반적 processing과정이고 오른쪽은 추천하는 사용툴이다. 기능적 연결성을 확인하기 위해서 fMRI 시계열 데이터를 사용한다. Brain network를 구성하기 위해 ROI를 선택하여 node로 지정한다. 각 노드의 representative한 fMRI BOLD(Blood Oxygenation Level Dependent) series 데이터를 구하기 위해서는 전체 노드안에 있는 데이터들의 평균을 구하거나 SVD(Singular Value Decomposition)을 실행한다.

이후 노드간의 연결성을 확인하기 위해서 두개의 ROI에서 나온 BOLD 시계열 데이터들을 대상으로 pairwise correlation을 진행할 수 있다. 혹은 partial correlation, mutual information, coherence, Granger causality등 다양한 Functional Connectivity Metric을 이용할 수도 있다. 본 논문에서는 pairwise correlation으로 FC측정을 하였고 이후 Fisher’s transformation과 같은 transformation이 적용되어 distribution property를 개선시킨다. 그렇게 transformed된 데이터는 subsequent analysis에 사용될 수 있다.

4. Structural Brain Network Construction

구조적 뇌 네트워크는 해부학적, 그리고 생리학적 perspective에서 뇌의 구조가 기능에 어떻게 영향을 미치는지 알 수 있게 해준다.

위의 도표는 뇌의 구조적 네트워크를 구성하는 pipeline을 보여준다. 전처리 과정은 dMRI 데이터의 eddy current로 유도된 왜곡 제거, brain extraction, 그리고 co-registration between diffusion and structural image가 있다. 이후 모델링 방법이 적용되어 local diffusion pattern을 reconstruct한다. 본 논문에서는 probabilistic tractography로 분석을 이어간다. 이는 우선 fiber orientation을 예측하고 fiber의 불확실성을 고려해서 diffusion path probability map을 구성하는 방법이다.해당 방법은 노이즈에 강건하고 불확실한 영역까지 다룰 수 있게 해주어 fiber orientation의 불확실성을 고려할 수 있게 만든다. 구조적 네트워크를 구성하기 위해 각 노드 pair에 대한 structure connectivity를 두 region을 잇는 fiber tracts의 empirical probability에 기반하여 설정한다.

Connection Network Construction에 관한 mainstream 방법론을 제외하고도 edge들을 형성하는 여러가지 방법들이 존재한다. 그리고 최근 study들에 따르면 기능/구조의 combined neuroimaging modality가 한개만 활용하는 것보다 효과적이라고 한다. 왜냐하면 complementary한 정보를 모두 다 exploit 할 수 있기 때문이다.

5. GNN BASELINES FOR BRAIN NETWORK ANALYSIS

Identity : GNN을 뇌 네트워크에 적용하는 과정은 ROI feature의 initialization - forward pass(message passing - pooling)으로 이루어진다. 그리고 학습된 graph-level representation은 뇌 관련 질환 분석에 사용될 수 있다. 우선 모든 노드에 대해서 unique한 one-hot vector가 초기화된다. Node들의 k-hop neighbor들을 저장함으로써 노드들의 상대적인 위치를 학습시킬 수 있다. GNN의 첫번째 linear layer의 parameter들이 랜덤하게 설정되기 때문에 실질적으로 노드들의 초기값은 랜덤하게 설정된다고 생각할 수 있다.

Eigen : ROI들 간의 연결 강도 정보를 담고 있는 matrix에 eigen decomposition을 실행하여 top k개의 eigven-vector들만이 각 노드의 k-dimensional feature를 생성하는데 사용된다. k의 값은 grid search를 통해 탐색된다. 이를 통해 차원 축소 효과와 인접한 위치의 뇌들을 grouping하고 낮은 차원에 임베딩하는 목적을 가진다.

Degree : 각 노드의 degree값은 node feature처럼 사용되고 1차원 정보이다.

Degree Profile : Degree와 관련된 정보를 concatenate하여서 xi 변수에 저장한다.

Connection Profile : 각각의 node에 해당하는 edge weight matrix를 초기 node feature로 사용하여 활용한다. 그리고 edge는 뇌 네트워크에 있는 다른 노드들의 모든 정보를 다 내포하고 있다.

6. Message Passing Mechanisms

GNN은 node representation을 iterative하게 update한다. 이는 local area에 있는 노드들과의 Neighbor Feature Aggregation을 통해 이루어진다. Message Passing 단게에서 node vi는 주변 neighbor 모두로부터 message를 받고 그리고 모든 message들은 sum function을 통해 합쳐진다. 아래 수식이 이를 보여준다.

Concatenated된 aggreagated 메세지들은 updata function u에 들어가서 l+1 level의 h 값에 대입된다. Message passing mechanism은 permutation equivariance(graph는 order가 없기에 permutation이 보장되어야 한다)와 local structure에 대한 inductive bias를 잘 이룬다. 또한 새로운 그래프에 대해 generalization이 잘 이루어진다.

논문에서는 다양한 메세지 함수 M의 영향을 조사하고 있다.

  • Edge Weighted :

주의해야 할 점은 m_ij의 notation이 직관적이지 않게 i에서 j로 가는 것이 아니라! j에서 i로 가는 것이라는 점이다. h_j가 w_ij와 곱해져서 노드 v_j에서 노드 v_i로 가는 message가 구성된다. 그리고 위의 m_ij가 standard한 GCN의 message passing implementation이다.

  • Bin Concat :

Edge weight w_ij를 T개의 bucket중에 하나로 할당한다. T는 grid search로 탐색되며 동일한 bucket 안에 포함된 edge들은 같은 parameter b_t를 사용한다. h_j와 b_t를 concatenate하여 MLP(Multi Layer Perceptron)에 통과시킨다. 위의 edge weight에서 dot product를 했다면 여기서는 그 역할을 concat+mlp로 대신한 것이다. 이러한 bin concat 방식은 비슷한 strengths의 region connections들을 cluster하는 걸 도와준다.

  • Edge Weight Concat :

Edge weight 방식도 위의 bin concat과 유사하나 node feature dimension을 곱한다는 점에서 차이를 보인다. 이를 통해 edge weight를 node feature의 dimension과 동일한 수준으로 스케일링 하고자 하는 의도에서 생긴 것이다.

  • Node Edge Concat :

마지막 time step에서의 brain region representation 정보를 남겨두는 것이 유의미한지를 판별하기 위해 h_i와 h_j 모두 남겨두었다. 이러한 design은 GNN의 over-smoothing 문제를 완화시킬 수 있다(over-smoothing이란 GNN의 고질적인 문제로써 노드들간의 feature distance들이 너무 가까워져 몇번의 layer convolution 이후 구별이 불가능해지는 상태를 일컫는다).

  • Node Concat :

Node Edge Concat과 유사하나 weight를 제거하였다.

7. Attention-Enhanced Message Passing

Attention은 human cognitive system이 선택적으로 주의를 집중하는 인지 과정에서 착안한 아이디어이다. 선택적인 중요한 부분에만 집중하기 때문에 방대한 정보를 처리하여야 할 때 유용하다. 고전적인 Graph Attention Network에서는 edge attribute를 고려하지 않는다. 그러나 뇌 네트워크의 경우 두 region의 correlation은 중요한 biomedical 정보를 포함하고 있기에 graph-level task에 도음이 될 수 있다. 따라서 본 논문에서는 다양한 attention-enhanced message passing mechanism을 시도한다.

  • Attention Weighted : Original GAT 모델이다. 즉, edge attribute들을 고려하지 않는 일반적 그래프이다.

  • Edge Weighted Without Attention :

    이는 Attention Weighted 모델에서 edge weight를 고려하도록 수정된 방법이다. Node v_j의 representation인 h_j, edge weight w_ij, 그리고 attention score a_ij를 모두 곱해서 message m_ij를 구성한다.

  • Attention Edge Sum :

attention과 weight를 합쳐서 impact factor로 활용한다.

  • Node Edge Concat Without Attention

  • Node Concat Without Attention

위의 모델에서 weight를 제외한 버전이다.

8. Pooling Strategies

GNN의 두번째 단계에서는 pooling이 이루어진다.

Pooling 함수 R은 그래프 안에 있는 node vector들 안에서 이루어지고 node vector들의 permutation에 영향 받지 않는다(invariant). 해당 논문에서는 세 가지 basic global pooling operator들을 다루고 있다. Pooling 방법은 graph Gn에 대한 graph-level representation을 계산하는 것이다.

  1. Mean Pooling : node feature들의 평균값으로 이루어진다.
  1. Sum Pooling : node feature들의 합으로 이루어진다.
  1. Concat Pooling : node feature들의 concatenation으로 이루어진다.

3. A high-fidelity, Machine-Learning enhanced queueing network simulation model for hospital ultrasound operations

1. Introduction

중국 선전의 Luohu Hospital의 초음파 센터에 대한 대기열의 길이, 대기 시간, 체류 시간을 예측하기 위한 모델 개발. 초음파 센터는 특히 산부인과와 관련하여 수요가 높으며, 하루 안에 의사의 진찰, 초음파 검사, 검사 결과를 바탕으로 한 재 진찰 등의 과정이 이루어져야 하므로 체류 시간을 정확히 예측하는 모델을 통해 환자들의 경험을 향상 시킬 수 있음.

초음파 센터는 multi-class, multi-pool parallel-server queue의 구조로 모델링을 진행함. 이때, heterogeneity, complicated patient routing, time-dependency 등의 특징들이 나타남.

heterogeneity: 각각의 환자들 마다 다양한 종류의 test가 필요하며, 검사 장비 또한 일반적인 것과 특별한 검사를 위한 것 등으로 나뉘어져, 각각의 검사들의 service time이 high variance를 가짐.

complicated patient routing: 서로 다른 환자들과 검사실들 사이의 경로가 매우 복잡하며, 특히 경로 설정에 대한 명확한 기준이 없으며(간호사의 재량에 의존) 모든 검사실이 계속 열려있지 않다는 문제가 있음(휴식 시간 등의 이유).

time-dependency: 대부분의 healthcare system에서 arrival rate는 높은 time-dependency를 가지며, service rate 또한 이른 아침보다 늦은 아침에 더 빨라지는 등의 경향성을 가짐.

따라서, 적절한 경로 모델을 만들어야 하고, 환자와 server의 분류를 적절한 수로 진행하며, server unavailability 문제를 잘 고려하여야 함.

Stochastic queueing network의 key modeling components를 보정하기 위해 머신 러닝 기법을 사용함. 머신 러닝 기법은 모델의 정확도 향상에 기여를 하며, queueing 기반의 모델은 설명력 등을 제공함.

Two-level patient routing component: 첫 단계에서는 환자들을 보낼 검사실 그룹을 선택하고, 두 번째 단계에서는 해당 그룹 내에서 환자를 보낼 검사실을 선택하는 방식으로 two-level hierachical routing policy를 개발함. 각각의 단계에서 머신 러닝 기법을 적용하였으며, 첫 단계에서는 환자의 medical factor를 고려하고, 두 번째 단계에서는 검사실들의 대기열 길이와 검사 가능 여부 등을 고려함.

Clustering-based patient and room classification: 환자들과 server들을 분류함. 환자들은 해당 환자들이 받아야 하는 검사 items와 특별한 검사를 받아야 하는지 등의 기준으로 분류를 진행하며, server들은 실제적인 확률과 opening time, service time 등으로 분류를 진행함.

Estimating server availability: 검사실을 열고 닫는 시간에 대한 것과 중간 중간의 검사실 휴식 시간을 고려해야 함. 첫 번째 사항은 단순히 data를 통해 확인할 수 있지만, 두 번째 사항은 data를 분석하여 gap time을 찾아내야 하며, 이 gap time 중 진짜 쉬는 시간과 환자의 이동 시간을 잘 분류하고 결합하여야 함.

이렇게 학습된 model을 통해 실제로 환자들의 경험을 좋게 발전시킬 수 있었으며, 이를 통해 좀 더 최적화된 scheduling도 가능해짐. 또한 개선된 schedule로 인해 환자들의 분포가 변화되었을 때 이를 input으로 하는 새로운 model의 학습도 가능함. 이는 queueing-based model의 장점으로, black-box 머신 러닝과 비교하였을 때 이점이 있음.

3. Simulation Model

모델은 multi-class, multi-pool parallel-server queueing system이며, (1) patients arriving, (2) checking room availability, (3) routing patients to the specific room, (4) patients receiving service and leaving the system after service completion의 4단계로 구성됨. 논문 속 모델은 (3) 단계에서 two-level 머신 러닝 기법을 적용했다는 것과, (4) 단계에서 환자의 service process에서 랜덤한 break time을 추가했다는 차별점이 있음.

(1) item type, age group, gender를 기준으로 환자를 총 75개의 서로 다른 그룹으로 분류하여 각각의 환자 그룹은 independent time-nongomogeneous Poisson process를 따르도록 모델링을 진행함. Arrival rate는 실제 데이터를 참고함.

(2) room opening 패턴은 평균적인 비율로는 estimate가 어려움. 그런데 opening 패턴이 각각의 day들과 강한 상관관계를 가지므로, each day 마다 패턴을 estimate함.

(3) 첫 level에서는 4개의 room types 중 환자를 배정할 한 가지 타입을 결정함. 이후 두 번째 level에서는 해당 type의 검사실들 중 어느 검사실에 환자를 배정할지 결정함. 첫 level에서의 할당은 medical 관련 내용이 강하게 작용하며, 두 번째 level에서의 할당은 대기열의 길이와 room availability 등의 요소가 강하게 작용하므로 이와 같은 two-level structure가 적절함.

첫 level에서의 지도 학습 분류 문제를 해결하기 위해 multinomial logit model, random forest, gradient boosting tree, neural network 등을 test 하였으며, random forest가 가장 좋은 정확도를 보여 선택함. 이때 feature는 age, item type, arrival hour, weekday, queue length(각 그룹의 대기열 길이의 합), numServer(각 그룹의 전체 검사실들 중 open한 검사실의 수)를 사용함.

두 번째 level 역시 지도 학습 분류 문제이며, 첫 단계와 마찬가지로 여러 방법을 test 했을 때 random forest가 가장 좋은 성능을 보임. feature 또한 첫 level과 거의 유사하지만 queue length가 전체가 아닌 각각의 검사실에 대한 대기열 길이 이며, numServer는 각 검사실의 open 여부를 binary로 표현했다는 차이가 있음.

(4) service time은 실제 data를 추출하여 사용하였지만, simulation model에서는 random server break time과 patient walking time을 추가함. 실제로 병원의 모습을 관찰하였을 때, 한 환자의 service를 끝냈다고 해서 바로 다음 환자를 부르는 것이 아니라 자재들을 정리하고 기기를 정비하는 등의 약간의 break time이 존재함. 이러한 break time에 대한 분포를 data에서 추출하여 각각의 room type과 day의 각 hour에 대하여 fitting을 진행함. 비슷한 방식으로 walking time 또한 각각의 room type과 day의 각 hour에 대하여 fitting.

4. Model Validation

첫 level에서 4가지 type의 rooms에 대해 실제 데이터와 비교했을 때 simulation model이 잘 작동함. relative difference는 0.60% ~ 22.55% 사이에 위치함. 두 번째 level 또한 simulation model이 잘 작동하며, relative difference는 0% ~ 15.31% 사이에 위치함.

새롭게 만든 simulation model과 JSQ(Join-Shortest-Queue) routing multi-class queue model을 비교하였을 때, differences in mean queue length & waiting time distributions에 있어서 눈에 띄는 차이가 존재함. Kolmogorov-Smirnov difference 역시 눈에 띄게 감소함. 전체적으로 새로운 model이 기존의 model에 비해 더 정확한 예측 성능을 가진다고 할 수 있음.

5. Conclusion

많은 health service systems에서 기존의 modeling-based analysis는 inflexible하여 여러 한계가 존재함. 이에 복잡한 특징들을 capture하고, 더 정확한 예측을 할 수 있는 high-fidelity simulation model이 필요함. 이러한 model을 통해 병원의 매니저들은 더 나은 결정을 할 수 있음.

4. Mixture of Input-Output Hidden Markov Models for Heterogeneous Disease Progression Modeling

1. Abstract

질병 진행 모델링에서 특히 어려운 점은 질병의 이질성(heterogeneity)과 환자에게서 나타내는 질병의 징후를 모델링 하는 것이다. 현재 존재하는 접근방식은 하나의 질병 진행 특징만이 존재한다고 가정하는 것인데, 이런 방식은 파킨슨 병과 같은 신경 변성 질환에 대해서는 맞지 않다.

해당 연구에서는 다중 질병 진행 패턴(multiple disease progression dynamics)을 발견할 수 있는 계층적 시계열 모델을 제안한다.

제안된 모델은 환자의 건강 상태에 대한 임상의의 평가와 처방된 약물들을 고려하는 input-output hidden Markov model의 확장된 형태다. 고안한 모델의 장점을 파킨슨 병에 대한 종단 자료를 이용하여 시각화했다.

2. Introduction

질병 진행은 시간에 따른 질병의 진화를 의미한다. 질병의 시간적 특성을 모델링하는 것은 과학적 발견(ex : 환자가 전형적으로 겪는 단계를 밝혀냄으로써 질병이 어떻게 나타나는지를 이해한다.)과 임상적 의사 결정(ex : 환자가 속한 질병의 진행 단계를 확인하여 환자의 건강 상태를 평가한다.)을 포함한 다양한 분야에 유용하게 쓰일 수 있다.

데이터에 시간적 관계가 존재하기 때문에 질병 진행 모델링에 있어서 확률적 시계열 모델의 활용은 자연스러운 선택이다. 하지만 확률적 시계열 모델은 한계점이 존재한다.

  1. 가용할 수 있는 데이터가 제한적이다.
  1. 결측치와 같은 데이터 문제
  1. 해석가능성(interpretability)의 필요
  1. 알츠하이머(AD)와 파킨슨 병(PD)과 같은 병이 지니는 근본적인 이질성을 반영하지 못한다.

이런 문제들의 전형적인 해결방안은 HMM(hidden Markov Models)를 사용하는 것이다.

HMM은

  1. 적은 데이터로 학습할 수 있고,
  1. 원칙적인 접근 방식으로 결측치를 다룰 수 있으며,
  1. 해석가능한 모델이다.

하지만 대부분 존재하는 HMM은 각 환자들이 동일한 상태 전이 패턴(Transition probability)을 가진다고 가정한다, 이는 질병 진행 과정의 이질성(heterogeneity)을 무시하는 가정이다.

이질성 질병 진행 모델링의 필요성은 질병 아형(disease subtype) 분류 작업에서 강조된다. 질병 아형 분류는 동일 질병 내에서 특징이 유사한 환자들을 그룹짓는 작업이다. 이를 통해 개인화된 치료 의사결정을 쉽게 내릴 수있다. 질병 아형 분류는 자폐증, 심혈관 질환, 파킨슨 병과 같이 이해하기 어려운 복잡한 질병에 대해 유용하다. 아형(subtypes)의 발견은 과학적 발견이나 임상적 의사결정에 도움을 준다.

전통적으로 질병의 아형 분류는 하위 그룹의 존재를 알아챈 임상의에 의해서 행해졌다. 최근에는, 가용한 의학 데이터와 계산 자원이 늘어남에 따라서 질병 아형을 발견해내는 데이터 기반 접근 방식을 적용할 수 있게 됐다.

예를 들어, 2005년에 PD에 대해 4가지 아형을 발견했는데, 해당 발견에 사용된 k-means clustering 기법은 데이터 속의 복잡한 패턴을 포착하기에는 제한적이었다. 2015년에는 혼합 모델(mixture modle)을 이용한 보다 정교한 접근 방식이 개발되었는데, 해당 모델은 질병 아형 분류와 무관한 변동성에 대해 강건했다. 하지만, 제안된 모델은 임상 방문에서 시간적 관계를 고려하지 않았다.

본 연구에서는 동일한 질병 상태 전이 확률을 모든 환자들이 공유한다는 HMM의 가정을 완화한다. 대신에, 질병 진행 모델링에 대해 계층적 HMMs(mixture of HMMs, 즉 mHMMs : HMMs와 환자의 그룹 수준 유사성을 명시적으로 모델링할 수 있는 변형된 HMMs의 혼합)의 사용을 제안한다.

mHMMs가 다른 분야에서 HMMs을 outperform한 것을 보고 사용해보게 되었다.

3. Contributions

이 연구는 (우리가 알기로) 질병 진행 모델링에 mHMMs를 적용하는 첫번째 시도다. 특히 mixture of input-output HMMs(mIOHMMs)가 질병 진행 모델링에 IOHMMs보다 적합하다는 것을 알았다. 왜냐하면 mIOHMMs가 다중의 질병 진행 패턴을 발견할 수 있기 때문이다. 게다가 mIOHMMs외에도 mixture of personalized HMMs (mPHMMs) and mixture of personalized IOHMMs (mPIOHMMs)을 비롯한 다양한 HMM의 변형을 개발했다.

7. Multi-modality machine learning predicting Parkinson’s disease

Abstract

(대충 정확한 예측에 머신러닝과 multimodal data의 결합이 중요하다는 내용)

선행 연구를 기반으로, Multi-modal하게 파킨슨병 위험도를 예측하고 머신러닝(GenoML)을 활용해 다중오믹스 기반 예측 모델을 발전시키고자 하였다. 주요 변수를 조사하고, 가설 무관 질병 관계도를 세웠으며, 약물-유전자 상호작용을 조사하였다. PPMI 데이터에 자동화 ML을 실시하여 최적의 알고리즘을 선택한 뒤 전체 PPMI 데이터로 다시 알고리즘을 조정하였다. 이후 알고리즘을 PDBP(Parkinson’s disease biomarker program) 데이터로 평가하였다. 최초 모델의 AUC는 89.72%, 조정한 모델의 외부 데이터(PDBP)에 대한 AUC는 85.03%였다. 분류를 위한 threshold를 최적화하는 것으로 예측 정확도와 기타 지표를 향상시킬 수 있었다. 최종적으로, PD와 연관된 gene community를 파악하기 위해 관계도를 생성하였다. 데이터 modality를 결합하는 것이 하나의 biomarker를 사용하는 것보다 우수한 것으로 나타났다. UPSIT(펜실베니아 대학 후각 테스트)과 PRS(Polygenic 위험 점수)가 모델의 예측 능력에 가장 큰 기여를 했으나 이들의 정확도는 더욱 많은, 작은 범위의 전사 과정과 SNP(Single nucleotide polymorphism, 단일염기다형성)등에 의해 보완된다.

Introduction

(대충 진행형 신경퇴행성 질환은 미리미리 진단하고 치료하는 게 중요하다는 내용)

오픈소스 auto-ML인 GenoML을 응용해서, 데이터 기반 가성비 좋은 초기 진단을 하고자 함. Multimodal genomics(유전체학)과 실제 데이터를, 민주적인 차원에서 양적 분석하는 맥락으로.

의생명계열 연구자들은 초기 진단과 잠재적 고위험군의 원격 진단을 위해 현재 두 가지 과학적 발전을 융합하는 중; 수많은 데이터와, 이를 최대한 활용할 수 있는 자동화된 머신러닝 파이프라인과 인공지능.

CSF biomarker, imaging, RNA 등등… 이 중 우수한 분류 성능을 보인 시도들도 있었지만, 이번 연구의 목표는 “가성비 좋고, 쉽게 접근 가능한 데이터(원격으로 생성 가능하거나, 기존의 biobank 데이터를 활용해서 추가적인 병원 방문이나 값비싼 검사 없이도)를 활용한 모델을 구축”하고자 함. 환자가 증상을 느끼기도 전에 미리 예측/진단하여, 가성비 좋게 미리미리 개입할 수 있도록. 어떻게? Multimodal 모델의 강점은 특정 modality가 다른 modality에 비해 더욱 잘 예측하거나, 상태를 control할 수 있다는 점임. 본 연구에선 datatype의 다양성을 이용해 전반적인 민감도와 정확도를 향상시켰다. 최종 multimodal model에서는 정확도 85.56, 균형정확도 82.41, TPR 89.31, TNR 75.51로 single modality 모델에 비해 우수하였음. 또한 cross validation으로 얻어진 AUC의 분포를 비교하여 multimodal 모델이 임상/인구통계학적 데이터 모델에 비해 일관되게 우수한 성능임을 확인하였음. 특히 균형정확도의 증가는 큰 의미를 갖는데, 예측 범주의 불균형이 심한(실제로 파킨슨 병인 경우가 매우 적기에) 데이터이기 때문.

Benefits of using machine learning to construct multimodal prediction model

기존의 회귀 기반 접근에 대비되는 머신 러닝 기반 접근의 장점 중 하나는 파라미터를 조정하고, 비선형적인 변수 간 관계를 반영할 수 있다는 것. (대충 AdaBoost 모델 계속 튜닝해서 우수한 성능 지표가 나왔다는 내용) 또한 post hoc(사후분석) 결과를 토대로 최적화한 결과 더욱 우수한 성능을 나타낼 수 있었음. 튜닝 후 최적화된 모델을 gold standard로 잡았고, 두 cohort의 크기를 맞추는 대신에 Youden index로 최적의 threshold를 선정함. 전반적으로 threshold 최적화를 통해 일반적인 분류 성능을, minimal한 계산 비용으로 향상시켰음.

Smell identification test and polygenic risk scores contribute most to predictive performance

51개의 SNP, 418개의 protein-coding transcript와 기타 인구통계학적 변수를 포함해서 모델을 구성하였고, SHAP 분석 결과 UPSIT과 PRS가 예측력에 가장 큰 기여를 하는 것으로 나타남. 낮은 UPSIT 점수는 더 높은 확률로 파킨슨병에 해당하여, 선행 연구들과 같은 결론. 또한 서로 다른 유전적 형질의 방향성이 균일하지 않은데, 특정 유전자의 overexpression이 건강에 도움이 되기도, 해를 끼치기도 하는 것을 확인. 성별 변수가 데이터셋에 포함되어 있는데, training data의 case와 control에 balanced distribution이 존재하였기에 feature selection에서 탈락한 것으로 보임. 가장 우수한 성능을 보인 모델들은 서로 다른 변수를 사용.

Gene expresion network communities and drug enrichment analysis

파킨슨병이 양성인 경우의 RNA 염기서열 데이터를 통해 Gene expression network communitiy를 생성. 이 Network는 일종의 pathway인데, 유전자의 발현과 파킨슨병 발병 사이에 강한 연관이 존재한 유전자들이고 community는 이들과 closely related gene들의 부분집합임.

Discussion

이제는 성능의 향상을 위해 multiple data modalites를 사용하는, Multimodal 모델이 가능. 기존의 보행이나 운동, 수면 등의 데이터를 사용한 연구들과는 다르게 전사체학, 유전학, 진단 데이터 모두를 ML을 프레임워크로 삼은 하나의 모델에서 사용하고자 함. 특히 기존의 연구들을 적은 부담으로 보완할 뿐 아니라, transfer나 앙상블과 같은 방법으로 결합할 수 있다는 점에서 잠재력이 있음. 앞선 연구들에서 UPSIT이 가장 예측력이 뛰어난 것으로 나타났고, 이를 기반으로 시작한 본 연구의 핵심 목표는 (직접 진료를 받지 않고도) 원격으로 획득 가능하거나 biobank sample에서 쉽게 얻을 수 있는 데이터를 기반으로 모델을 세우는 것이었음. 특히 아래 Mei 등의 논문에서 분석한 기존의 ML 기반 파킨슨병 진단 연구들과의 본 연구의 차이점은 modality에 집중하고 공공에서 접근 가능한 데이터를 기반으로 하였다는 것.

Data 기반 분석을 통해 biobank와 healthcare system에서 정확하고 신속한 진단이 가능. 본 연구에서는 강건한 진단 전후 PD 모델 뿐 아니라 새로운 network community 역시 세울 수 있었음. 본 연구의 모델은 투명한 open science 프레임워크를 통해 전 과정이 이루어졌고, modality들이 서로를 보완해주는 것이 강점.

실제 진단 후 선별을 대체할 수는 없지만, 대규모의 biobank나 연구 대상들 중 고위험군 파악에 큰 도움을 줄 수 있음. 다만 고위험군을 대상으로 하는 연구에서 다른 질병들과 극초기 파킨슨병을 올바르게 분류할 수 있는 능력이 있는지는 추가적인 연구가 필요함. 한편 고령층에서 파킨슨병의 유병률은 2%정도로 추정되고, 모델에 따르면 해당 유병률에서 False omission rate가 0.34%, Positive predictive value가 8.75%. 모델에서는 실제 유병 환자의 10배 정도를 고위험군, 혹은 잠재적인 파킨슨병 환자로 판단할 것이며 이는 모델이 추가적인 진단을 필요로 하거나 우선순위에 둬야 할 집단을 분류하는 역할에 적합하다는 것을 뜻함(Specifity는 조금 낮더라도 Sensitivity가 그만큼 높기에)..

모델의 다른 강점은 환자군과 대조군을 높은 균형정확도(Balanced accuracy)로 묘사 가능한 점. PDBP와 PPMI는 차이가 존재하는데,

PPMI : 최근 진단, 치료 이전의 환자군, 진단 1년 이내 medication X

PDBP : 더 현실적인 남녀비율, 5년 내로 진단 medication 여부는 신경 X

이로 인해 두 데이터에서 개별 모델의 성능이 약간 다르게 나타남. 전사체학, 유전학 모델이 PDBP에서 underperform, 한편 clinico-demographic 모델은 PDBP에서도 준수한 성능. 또 다른 강점은 다른 데이터셋에 적용할 수 있다는 점.

UPSIT 점수가 모델이 대상을 파킨슨병 환자로 판별하는 데 가장 중요한 변수였으나, 후각 퇴행은 파킨슨병만의 문제가 아니고 신경퇴행성 질환의 전반적인 증상임에 유의해야 함. 아무튼, 모델의 의사 결정에 거의 절반 정도 영향을 끼치고 축복이자 저주라고 볼 수 있음.

다음 문단은 어떤 염색체/유전체가 중요한 변수로 나타났다는 내용인데… 우리 관심사와는 약간 벗어난 듯 하여 생략. 다만 genetic의 effect들은 앞선 연구 결과들과 비슷하게 나타났고, 모델에 포함된 genetic이나 PRS(다유전자 위험 점수)는 데이터보다는 파킨슨병 관련 선행 지식에 기반하였기에 어느정도 과적합이 일어났을 수 있음(bias!).

Feature selection을 통해 간단하게 선택된 변수들로 네트워크 커뮤니티를 만들고, 이를 통해 다시 어떤 약물을 사용해야 할 지 알 수 있는 것 역시 강점.

연구의 보완점은 표본의 다양성이 부족했던 점. 현재의 연구 결과로는, 유전적 예측 모델이 유전 그룹(인종적인? 이야기인듯)에 따라 다른 결과를 도출하기도 함. 또한 PPMI에서 도출된 결과(초기 모델)을 validate 하기 위한 최적의 데이터가 없다는 것도 단점. 다른 파킨슨병의 주요 지표인 REM 수면 중 행동, 변비 등이 분석에 포함되지 않았음. 다만 PPMI 데이터의 확장이 추후 연구에 도움이 될 것으로 예상.

Methods

Study participants

임상적, 인구통계학적, 그리고 게놈 단위의 DNA-RNA 염기서열 데이터는 PPMI와 PDBP에서 획득. PD 환자군과 신경퇴행성 질환이 없는 대조군. 학습과 검증 과정의 최소 80%에서는 사용 가능한 변수들을 유지하려 하였고, 기존 연구에서 포함되었던 변수들과 전반적인 omic data를 우선하였음(앞선 연구에선 유전체학 위주 였기에 *omic data는 ~~학 데이터를 의미). RNA signature는 질병의 단계에 따라 변화하지만, baseline의 RNA signature가 진단 시점과 가장 가깝기에 선택함. 학습용 데이터는 PPMI로, 앞서 언급했듯 진단 1년 이내의 약물이 투여되지 않은 개인들의 data이기에 선택. 다시 진단 결과 간에 충돌이 있는 환자군 제외, 이후 신경퇴행성 질환이 생긴 대조군 제외, 특정 인자(LRRk2와 GBA 변이 보유자) 관련 데이터 제외. 유럽 유전 그룹의 개인들에 대해 분석을 진행하였음. 해당하는 진단, 인구통계학적, 그리고 유전학적 데이터를 모두 보유한 참가자들의 데이터를 포함하였고 각 feature당 15% 이상의 결측치가 존재하는 경우는 제외.

진단, 인구통계학적 데이터의 간단한 요약은 Table 1과 같음. DNA 서열 데이터는 Illumina의 기술을 활용, RNA 서열 데이터는 Illumina NovaSeq 기술을 활용. 자세한 내용은 생략… RNA는 단백질 생성 코드를 기준으로 환자군과 대조군 사이의 p-value를 로지스틱 회귀분석으로 계산함. (←유전체학 관련 내용이 좀 어려움, 더 전문적인 지식 필요…)

Data Preprocessing

DNA 염기서열의 경우, Linkage disequilibrium(연결 불균형, 유전자간의 거리가 가까워 독립적인 recombination이 일어나지 않아 기대치 이상의 연관이 나타나는 현상)을 pruning한 10000개의 변종에서 10개의 주요 component가 계산. RNA의 경우에는 모든 단백질-생성 유전자의 read count/sample로 10개의 주요 component 집합을 또 생성. 유전 변이를 나타내는 잠재적 변수들은 이렇게 DNA에서 얻어진 10개 주요 요소에 맞춰 선형 회귀로 조정. RNA 서열 데이터에도 같은 방법 적용. 간단히 말해 모든 전사체학적 데이터를 잠재적인 교란자에 대해 correct 하였고, genotype dosage도 correct한 것이다. (모르겠으면 밑줄 친 내용만 보자) 이후 모든 연속형 변수를 평균 0, 표준편차 1로 정규화. 변수 조정과 정규화 후에는 extraTrees 분류기를 통해 PPMI 데이터셋에서 feature selection. 변수들 간의 상관관계가 존재하거나 중요하지 않은 변수를 제거.

PPMI 데이터셋을 70:30으로 나누어 변수 및 알고리즘 선택. 가능한 적은 변수를 사용하는(parsimonious)한 모델을 목표. Shapely 분석 결과 상위 5%의 변수들은 평균 correlation이 r^2 <5 %, 최대 36%로 나타남. 동일한 training과 testing data에 대해 알고리즘 별 성능을 비교하고, 최적의 알고리즘에 대해서는 다시 5-fold 교차검증으로 하이퍼파라미터를 조정. 성능을 향상시키고 bias를 감소시키는 것을 목표함. PPMI로 학습, PDBP로 평가(두 데이터셋은 각각 정규화함).

로지스틱 회귀분석, 랜덤포레스트, AdaBoost, GradientBoosting, SGD, SVC, MLP, K-NN, linear discriminant analysis, qudradic discriminant analysis, bagging, XGB의 12가지 방법론 사용. 변수 선택에서는 extraTrees. PPMI 데이터의 30%, 즉 검증용 데이터에서 AUC와 Balanced accuracy가 가장 좊은 알고리즘을 선택. AdaBoost의 1에서 1000 사이 predictor를 five-fold 교차검증 25번으로 하이퍼파라미터 튜닝. Feature selection 간 [1E-2~1E-8]까지의 p-value threshold combination을 기준으로 모든 조합들을 iterate. 또한 modality 간의 차이를 고려하여 각 modlaity마다 별개의 feature selection 절차를 거침. 따라서 7 X 7 = 49개의 iteration, 그리고 iteration마다 최적의 AUC를 가진 모델을 선정하였고 다시 AUC와 기타 성능 지표를 종합적으로 평가하여 최종적인 모델을 선택. 본 연구의 모델에서는 genetic data inclusion에 대해서는 1E-5의 threshold, transcriptomic data에 대해서는 1E-2의 threshold.

최종적으로 Youden’s J를 통해 최적의 cut off threshold를 찾음. 그리고 외부 검증용 Cohort에 모델을 fit한 뒤 다시 최적의 threshold를 탐색. PPMI를 기준으로 학습된 모델을, PDBP 데이터셋에 적용해 예측한 결과에 따라 Youden’s J를 기준으로 threshold를 설정하는 것으로 PDBP 데이터셋에서 더 좋은 성능을 낼 수 있음. (여기 잘 이해 안되면 의견 남겨주시면 다시 구체적으로 설명함)

각 feature의 영향은 SHAP을 통해 계산. (SHAP에 대한 추가적인 설명이 필요하다면 의견 남겨주시면 다시 설명…)

8. Interpretability and fairness evaluation of deep learning models on MIMIC-IV dataset

Intro

사망률 예측, 체류 기간 예측, 표현형 분류 등의 과업에 있어 데이터 기반의 딥러닝 방법론이 전통적인 방법론들에 비해 우월한 성능을 보이고 있다. 그러나 딥러닝 방법론은, 과거의 로지스틱 회귀분석이나 랜덤 포레스트 등과는 다르게 소위 말하는 블랙박스 알고리즘이기에 헬스케어와 같은 분야에서 쉽게 적용하기 어려운 것 또한 사실이다. 본 연구에서는 MIMIC 데이터셋에 대해 model interpretability, dataset bias, algorithmic fairness, 그리고 interpretability와 fairness 사이의 상호작용에 대해 알아보고자 한다.

Interpretability evaluation

Bias and fairness in machine learning

데이터셋이나 모델에 존재하는 편향으로 인해 알고리즘이 unfair하게 되어 특정 집단을 선호하게 될 수도 있다. 일반적인, 알고리즘이 편향되게 짜인 건 아니라는 가정 하에 이러한 편향은 데이터에 존재하는 편향 때문이다. 1) biased labels, 2) imbalanced representation, 3) data quality disparity가 편향의 주 원인이다. 현존하는 실시간 예측 방법들은 고전적인 통계 기법을 사용하므로, 최근의 ML 기법의 발전의 이점을 누릴 수 없음. 이에 이 논문은 gradient boosted 방법을 사용하여 clinical risk를 추정하는 BoXHED 방법론을 사용함. MIMIC-IV data set을 이용하여 모델을 학습시켰고, 평가를 위해 고전적인 통계 방법인 Cox 모델과 생존 데이터를 사용하는 deep learning 모델인 Dynamic DeepHit와 비교를 진행함.
